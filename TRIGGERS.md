# TRIGGERS.md — 触发词系统

> 本文档系统化定义 `org-design-toolkit` skill 的触发关键词、触发场景、误触发处理、跨 skill 协作规则。
>
> Claude 在用户消息中识别到下列触发词时，应主动调用本 skill。

---

## 一、触发原则

### 1.1 触发的本质

skill 触发 = **"用户表达的意图"** 与 **"skill 能做的事"** 强匹配。

`org-design-toolkit` 能做的：
- **核心动作**：生成分工矩阵、岗位矩阵、工作分析表
- **次要动作**：组织架构设计、部门职责梳理、岗位说明书、RACI 设计
- **不能做的**：组织诊断（前期）、绩效系统（后期）、薪酬体系（后期）、组织变革实施（落地）

### 1.2 三级触发强度

| 级别 | 含义 | 处理 |
|---|---|---|
| 🔴 **主触发**（强信号） | 用户明确说出 skill 核心产物名 | 立即调用，无需确认 |
| 🟡 **弱触发**（中信号） | 用户描述了与 skill 相关的场景，但未明确说产物名 | 先确认意图，再调用 |
| 🟢 **协作触发**（跨 skill 联动） | 用户在做组织咨询全流程，本 skill 是其中一环 | 检查上下游 skill 输出，主动衔接 |

## 二、主触发词清单（🔴 立即调用）

按"产物名 / 工具名 / 动作名"三类整理：

### 2.1 产物名（最强信号）

| 触发词 | 同义表达 |
|---|---|
| 分工矩阵 | 部门分工矩阵、岗位分工矩阵、RACI 矩阵、责任矩阵 |
| 工作分析表 | 岗位说明书、岗位描述、岗位职责书、Job Description / JD |
| 分工手册 | 部门手册、职责手册、责任手册 |
| 组织架构图 | 组织结构图、Org Chart、组织设计图 |
| 岗位职责 | 岗位职责清单、岗位职责书、职责描述 |
| 部门职责 | 部门职责清单、部门职责书 |
| 任职资格 | 岗位资格、任职条件、JD 任职要求 |

### 2.2 工具名

| 触发词 | 说明 |
|---|---|
| RACI | Responsible/Accountable/Consulted/Informed |
| RAI | 简版 RACI（R 主责 / A 审核 / I 参与） |
| 主责/参与/监督 | 中文版 RACI |
| AKR | Accountable + Key Result 矩阵 |
| OARP | Owner/Approver/Reviewer/Participant |
| 三定 | 定岗 / 定责 / 定编 |

### 2.3 动作名

| 触发词 | 同义表达 |
|---|---|
| 做分工矩阵 | 写分工矩阵 / 出分工矩阵 / 整理分工矩阵 |
| 出工作分析 | 出工作分析表 / 写岗位说明书 / 做 JD |
| 组织设计交付 | 组织设计输出 / 组织设计落地 / 方案共创 |
| 梳理职责 | 梳理岗位职责 / 梳理部门职责 / 职责归位 |
| 三定 | 三定方案 / 定岗定责定编 |

### 2.4 复合短语（最高匹配度）

直接命中以下短语，**100% 应触发**：

```
- "做一份 [企业名] 的分工矩阵"
- "帮我出 [企业名] 的岗位分工"
- "[企业名] 的工作分析表怎么写"
- "做 [企业名] 的组织设计"
- "[企业名] 部门职责怎么定"
- "[企业名] 三定方案"
- "RACI 怎么填"
- "分工矩阵的标准格式"
- "岗位说明书模板"
```

## 三、弱触发词（🟡 先确认意图）

下列词语 **可能** 指本 skill，但也可能指上下游 skill：

| 弱触发词 | 可能的意图 | 应澄清的问题 |
|---|---|---|
| 组织设计 | 本 skill 或组织诊断 | "是要做分工矩阵 / 岗位说明书，还是先做组织诊断？" |
| 组织架构 | 本 skill 或组织变革 | "是要做新架构方案，还是要分析现状？" |
| 组织变革 | 落地辅导阶段，不一定是本 skill | "是要设计方案，还是要执行变革？" |
| 部门设置 | 本 skill 或战略规划 | "是要梳理部门职责，还是要做部门战略？" |
| 岗位优化 | 本 skill 或绩效系统 | "是要梳理岗位职责，还是要调岗位 KPI？" |
| 流程梳理 | 可能是流程优化 skill | "是要做分工 RACI，还是要做流程图？" |
| 人才盘点 | 不是本 skill（人才盘点是 HR 侧） | 跳转到 HR / 招聘 skill |
| 写一份方案 | 太泛 | "是组织设计方案、绩效方案、还是别的？" |

### 弱触发的标准追问模板

```
我注意到您提到了 [弱触发词]。这可能对应几个不同的工作：
1️⃣ 分工矩阵 + 岗位说明书（本 skill）
2️⃣ 组织诊断 + 健康度分析（前置工作）
3️⃣ 绩效 / 薪酬 / 招聘体系（后续工作）

您具体是想做哪一项？
```

## 四、反触发词（🚫 不要触发）

下列场景**不应**触发本 skill：

| 反触发词 | 原因 | 应跳转到 |
|---|---|---|
| 组织诊断 / 健康问卷 | 是前期诊断，本 skill 是下游 | 组织诊断 skill |
| 战略目标 / OKR / KPI 设定 | 是绩效系统的工作 | 绩效设计 skill |
| 薪酬 / 工资 / 调薪 | 是薪酬体系 | 薪酬设计 skill |
| 招聘 JD 投放 | 是招聘工作（虽然 JD 来自本 skill）| 招聘 skill |
| 培训 / 学习计划 | 不是本 skill | 培训设计 skill |
| 文化建设 / 价值观 | 不是本 skill | 文化建设 skill |
| 经营单元 / 合伙人 / 股权激励 | 不是本 skill | 经营改善 skill |
| 流程图（BPMN / Swim Lane） | 流程优化工具 | 流程优化 skill |

### 反触发标准应答

```
您提到的「[反触发词]」更适合用 [推荐 skill 名] 来处理。
本 skill（org-design-toolkit）专注于"分工矩阵 + 工作分析表"的输出。
如果您也需要分工矩阵 / 岗位说明书，告诉我可以并行处理。
```

## 五、跨 skill 协作触发（🟢 主动衔接）

### 5.1 典型组织咨询全流程

```
[组织诊断 skill] → [本 skill: org-design-toolkit] → [绩效设计 skill] → [薪酬设计 skill] → [辅导落地 skill]
        ↓                       ↓                        ↓                       ↓
   诊断报告 + 一句话诊断    分工矩阵 + 工作分析表        OKR / KPI 体系          薪酬带宽 + 调薪规则
```

### 5.2 触发上游联动

当用户消息中 **同时** 包含本 skill 主触发词 + 以下关键词时，应**主动检查**是否有上游产出：

| 关键词 | 主动检查 | 处理 |
|---|---|---|
| 诊断报告 / 调研报告 | Glob 上游诊断文件 | 自动预填 strategy.one_sentence_diagnosis |
| 一句话诊断 | 同上 | 同上 |
| 组织健康问卷 | Glob 问卷文件 | 自动预填 strategy.key_levers |
| 产品流程图 | Glob 流程图 | 启动专项动作 7.1 |
| 双工厂 / 多地 | 询问详情 | 启动专项动作 7.3 |

### 5.3 触发下游联动

skill 完成后，**主动建议**用户下一步用什么 skill：

```
✓ 本次产出 8 份组织设计文件已完成。

接下来您可能想做：
1. 🎯 绩效系统设计（基于 15 张工作分析表中的 KPI 母表）
2. 💰 薪酬体系设计（基于 10 部门的岗位级序）
3. 📋 OKR 季度分解（基于战略目标 → 部门 → 岗位的三层穿透）

需要继续哪一项？
```

## 六、误触发处理

### 6.1 用户表达模糊时

**用户**："帮我做个组织架构"

**Claude 应答**：
```
您说的"组织架构"可能涉及：

A. 现状架构梳理（拉树形组织图）
B. 新组织架构设计（含分工矩阵 + 岗位说明书）⭐ 本 skill 主营
C. 组织变革方案（包含步骤 / 时间表 / 抵抗管理）

您是 B 吗？如果是，我马上启动 org-design-toolkit，10 轮对话 5 分钟出图。
```

### 6.2 用户企业规模不匹配时

| 企业规模 | 适用性 | 应答 |
|---|---|---|
| < 30 人 | 不建议 | "30 人以下企业用分工矩阵反而束缚活力，建议先做战略 + 核心岗位 3 张 JD 即可" |
| 30-50 人 | 边界 | "可以做精简版（5 部门 × 30 项 RACI）" |
| **50-500 人** ⭐ | 最佳 | 全功能使用 |
| 500-1000 人 | 可用 | "建议拆分'集团级 + 事业部级'两层做" |
| > 1000 人 | 不建议 | "已超出 skill 设计规模，建议用专业 HRIS 工具" |

### 6.3 用户已有矩阵需要修改时

**用户**："这是我们已有的分工矩阵，帮我改一下"

**Claude 应答**：
```
您是要：

A. 微调（改几个 A 岗 / 加几个事项）→ 直接修改文件
B. 整体重做（v1 → v2 增量迭代）→ 启动 org-design-toolkit 增量模式
C. 对标改写（参照另一家企业的格式）→ 用 RAG 模式

您是哪一种？
```

## 七、触发示例

### 示例 1：清晰主触发 → 立即调用

**用户**："做一份示例企业的分工矩阵和工作分析表"

**Claude**：✓ 立即触发，进入第 0 步 RAG 扫描。

### 示例 2：弱触发 → 先澄清

**用户**："帮我做组织设计"

**Claude**：
```
您提到的"组织设计"具体是：

1️⃣ 分工矩阵 + 岗位说明书（本 skill 主营）
2️⃣ 组织诊断 + 健康度分析（前置工作）
3️⃣ 组织变革落地方案

是 1️⃣ 吗？
```

### 示例 3：反触发 → 跳转

**用户**："帮我做绩效考核方案"

**Claude**：
```
绩效考核方案最好用绩效设计相关 skill。
不过，绩效的母表是"15 张工作分析表"中的 KPI 列。

您是否已有分工矩阵 + 工作分析表？
- 已有 → 直接进入绩效设计
- 没有 → 先用 org-design-toolkit 5 分钟出母表
```

### 示例 4：协作触发 → 主动衔接

**用户**："这是示例企业的组织诊断报告，下一步该做什么？"

**Claude**：
```
诊断完成后的标准下一步是「方案共创」，即输出：
- 分工矩阵（部门级 + 岗位级）
- 15 张工作分析表
- 组织设计预览

我现在用 org-design-toolkit 帮您出，10 轮对话 5 分钟完成。
诊断报告已读取，会自动预填战略 / 一句话诊断 / 发力点。

继续？
```
