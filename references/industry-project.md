# 项目型 · 子行业事项库

适用范围：建筑工程 / 系统集成 / 软件外包 / IT 咨询实施 / 装修 / 工程承包 等"按项目交付"业务。
典型企业规模：30-1000 人，年营收 5000 万 - 10 亿。

---

## 一、子行业特征

| 维度 | 项目型 | 与流程型制造业的差异 |
|---|---|---|
| 业务模式 | **每个项目独立** / 一次性交付 | 制造业是重复批量 |
| 收入特征 | **项目里程碑分期付款** | 制造业是订单一次回款 |
| 成本特征 | 人工成本为主（70%+） | 制造业是物料 + 人工 |
| 周期特征 | 项目周期 1 个月 - 3 年 | 类似但弹性大 |
| 客户结构 | 大客户（B端 / 政府） | 类似 |
| 风险特征 | **项目超期 / 超预算 / 范围蔓延** | 制造业风险类型不同 |
| 财务特征 | DSO 90-180 天 | 制造业更短 |

## 二、10 节点核心创利流程（项目型专用）

| # | 节点 | 行业标准时长 | 责任部门组合 |
|---|---|---|---|
| 1 | 商机识别 | 持续 | 市场 + BD |
| 2 | 售前方案与投标 | 7-30 天 | 售前 + 商务 + 技术 |
| 3 | 合同谈判与签约 | 14-60 天 | 商务 + 法务 + 财务 |
| 4 | **项目立项与组队** ⭐ | 3-7 天 | PMO + 资源部 |
| 5 | 需求确认与详细设计 | 30-90 天 | 项目组 + 客户 |
| 6 | **项目执行**（开发 / 实施 / 施工） | 1 月 - 2 年 | 项目组 |
| 7 | 阶段交付与验收 | 7-30 天 | 项目组 + 客户 |
| 8 | 项目变更管理 | 持续 | PMO + 项目组 |
| 9 | 终验与质保 | 30-90 天 | 项目组 + 售后 |
| 10 | 回款与结项 | 30-180 天 | 财务 + 项目组 |

## 三、推荐组织架构（弱矩阵 / 强矩阵 / 项目型 三选一）

### 推荐：强矩阵型组织（10 个部门 + 项目组）

| 部门 | 编制建议 | 关键岗位 |
|---|---|---|
| 总经办 | 2-3 人 | 总经理 / 总工 |
| 市场 + BD 部 | 5-15 人 | BD 总监 / 商机经理 |
| 售前部 | 3-10 人 | 售前总监 / 解决方案架构师 |
| 商务部 | 3-8 人 | 商务总监 / 商务经理 / 合同管理 |
| **PMO 项目管理办公室** ⭐核心 | 3-10 人 | PMO 主任 / 项目经理（PM）池 |
| **资源部 / 资源池** ⭐核心 | 含全部技术人员 | 资源总监 / 技术 leader / 工程师 |
| 研发部 / 技术中心 | 5-30 人 | 技术总监 / 架构师 |
| 售后服务部 | 5-15 人 | 售后总监 / 服务工程师 |
| 财务部 | 5-10 人 | 财务总监 / 项目财务 BP |
| 人力行政部 | 5-10 人 | HRD / 招聘培训 |
| **项目组**（动态）| N × 项目编制 | PM / 技术组长 / 工程师 / 实施 |

### 关键设计原则

1. **PMO 独立**——必须有独立 PMO 监督所有项目，敢于"红线预警"
2. **资源池 + 项目组双汇报**——工程师"行政上"属资源部，"业务上"属项目组
3. **PM 必须有项目盈亏责任**——不只是进度责任，必须考核项目毛利
4. **强矩阵 vs 弱矩阵选择**：营收 <1 亿用弱矩阵；1-3 亿用强矩阵；>3 亿用项目型组织

## 四、推荐 KPI 范式

| 岗位 | 核心 KPI 1 | 核心 KPI 2 | 核心 KPI 3 |
|---|---|---|---|
| 总经理 | 年度营收 / 净利润率 | **项目毛利率** ≥30% | 新签合同额 |
| BD 总监 | 新签合同额 | 商机转化率 | 大客户营收占比 |
| 售前总监 | 投标中标率 ≥30% | 售前响应时效 | 方案技术评分 |
| PMO 主任 | **项目准时交付率** ≥85% | 项目超预算 ≤10% | 客户满意度 ≥4.5 |
| 资源总监 | 资源利用率 70-85% | 关键岗位流失率 ≤8% | 技术认证人数 |
| 技术总监 | 重大技术故障 = 0 | 技术中台复用率 | 创新专利数 |
| 项目经理（PM）| **项目毛利率** | 准时交付 | 客户满意度 |
| 财务总监 | DSO ≤120 天 | 项目财务可视化 | 现金流安全 |

## 五、典型陷阱

1. **PM 无毛利责任** — 只考核进度，导致超成本严重
2. **资源池虚化** — 没有强力资源调度，工程师"包卖给项目组"形成小山头
3. **范围蔓延（Scope Creep）** — 必须有强变更管理流程
4. **客户拖延验收** — 必须合同约定"变更窗口 + 验收时限"
5. **回款依赖客户关系** — 必须建立分阶段付款 + 法律保障
6. **售前过度承诺** — 必须有"售前 vs 实施"承诺一致性闸门
7. **关键人风险** — 项目经理离职导致项目失控，必须有"双 PM 制"

## 六、推荐分析诊断重点

| 诊断点 | 关键问法 |
|---|---|
| PMO 成熟度 | 是否有独立 PMO？PMO 直接向谁汇报？过去 3 年项目超期 / 超预算比例？ |
| 项目盈利 | 平均项目毛利率？亏损项目占比？亏损项目根因？ |
| 资源利用 | 资源利用率？技术 leader 占比？关键人风险？ |
| 售前-实施衔接 | 售前承诺与实施落地差异？复盘机制？ |
| 变更管理 | 项目变更控制流程？变更带来的额外营收 / 损失？ |
| 客户回款 | DSO？分阶段付款占比？应收账款催收成功率？ |
