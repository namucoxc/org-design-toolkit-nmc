# prompts/dialogue.md — 2 轮极简对话模板（v2.0.5）

> v2.0.5 起本 skill 输入收敛到 **2 个核心输入**——「经营创利流程」+「行业」。其他字段由 RAG + 行业模板自动推导，咨询师只需 1 次 review 拍板。
>
> 整个流程 **5-10 分钟**，相比 v1.0 的 10 轮版（25-45 分钟）压缩 **75%**。

---

## 设计哲学

> **AI 推断 → 咨询师审定 → 一次拍板**

旧版（10/5 轮）让咨询师"输入很多 → 等 AI 拼"。新版（2 轮）让咨询师"输入最少 → 看 AI 推 → 改它的"。后者更符合咨询师的工作习惯：他们更擅长 **review 别人的草稿**，而不是 **从零填表**。

---

## 第 0 轮 · RAG 自动扫描（0 问题 · 30 秒）

启动后自动执行：

```bash
# 优先级 1：官方分工矩阵案例
Glob: **/{分工矩阵*,组织设计*,工作分析*}/**

# 优先级 2：客户既有材料
Glob: **/{诊断*,调研*,组织*,组织架构*,产品流程*,工作分析*}.{docx,pptx,pdf,xlsx,png,jpg}
```

自动加载并标 `[数]` 来源。**无任何材料也不阻断**，直接走第 1 轮。

## 第 1 轮 · 核心输入采集（2 问 · 1-2 分钟）

### 问题 1.1 · 经营创利流程

```json
{
  "question": "请提供客户的经营创利流程（任选一种形式）",
  "header": "创利流程",
  "options": [
    {
      "label": "上传文件 (Recommended)",
      "description": "PDF / 图片 / PPT / Excel——AI 自动解析节点+时长+责任部门"
    },
    {
      "label": "提供工作目录中的文件路径",
      "description": "如 ~/projects/客户A/产品流程.pdf"
    },
    {
      "label": "文字描述（任意格式）",
      "description": "如：打样(29天)→货期(15天)→合同→评审→采购→生产→品检→出货→回款（10 节点）"
    },
    {
      "label": "无流程图，用行业默认",
      "description": "skill 用所选行业的标准创利流程"
    }
  ],
  "multiSelect": false
}
```

**AI 处理逻辑**：
- 文件 → OCR + 表格识别 → 提取节点/时长/部门
- 文字 → NLP 拆解 → 节点列表
- 默认 → 加载 `references/[行业].md` 的标准流程

预期产出字段：`strategy.core_profit_flow[]` + `strategy.core_profit_flow_source`

### 问题 1.2 · 行业归类

```json
{
  "question": "企业属于哪个行业？",
  "header": "行业",
  "options": [
    {"label": "制造业-精品包装 (Recommended)", "description": "礼盒/项链盒/香水盒，16 节点流程"},
    {"label": "制造业-通用", "description": "机械/五金/注塑/电子组装，12 节点"},
    {"label": "制造业-外贸出口", "description": "出口占比 ≥30%，18 节点（含清关退税）"},
    {"label": "其他（化工/食品/连锁/项目型）", "description": "进入二级选择"}
  ],
  "multiSelect": false
}
```

若选「其他」，追问二级选择：

```json
{
  "question": "请选择具体子行业",
  "header": "子行业",
  "options": [
    {"label": "化工（含 EHS 部强制）", "description": "精细化工/涂料/试剂/日化"},
    {"label": "食品（含食安部强制）", "description": "食品加工/饮料/烘焙"},
    {"label": "零售连锁", "description": "餐饮/门店/服务连锁"},
    {"label": "项目型", "description": "建筑/系统集成/软件外包"},
    {"label": "多地协同叠加层", "description": "总部+工厂/集团+子公司，与上述任一组合"}
  ],
  "multiSelect": false
}
```

预期产出字段：`meta.industry` + 自动加载 `references/[行业].md`

## AI 自动推导阶段（无问题 · 30 秒 - 1 分钟）

第 1 轮完成后，AI 静默执行：

```
1. 解析创利流程图 → 得到 N 个节点 + 责任部门列表
2. 加载 references/[行业].md → 加载 60-80 项行业默认事项库
3. 合并流程节点 + 行业模板 → 公司级事项清单
4. 从 references 推导组织架构（10 部门）+ 岗位清单（15 张）+ KPI 范式 + 产出表单库
5. 识别灰色地带：企业简称、A 岗归属、具体人名、薪酬数字
6. 整合为完整 JSON schema 草稿（v0.1）
```

## 第 2 轮 · schema review 拍板（1 问 · 3-7 分钟）

### 屏幕显示完整 schema 摘要

```markdown
═══════════════════════════════════════════════════════════
schema 草稿（v0.1 自动推导）· 待 review
═══════════════════════════════════════════════════════════

【输入溯源】
  · 创利流程：产品流程.pdf 已解析 → 16 节点（总时长 124 天，瓶颈 4 节点）
  · 行业：制造业-精品包装 → 加载 references/precious-packaging.md

【自动推导内容】
  ⬇ 组织架构（10 部门）
    总经办 / 业务部 / 跟单部 / 研发部 / 生产部 / 采购部 / 品管部 / 仓储部 / 财务部 / 人力行政部

  ⬇ 岗位清单（15 张工作分析表）
    高管 7：总经理 / 业务总监 / 研发总监 / 生产总监 / 采购总监 / 财务总监 / 人力行政总监
    中层 8：业务大区经理 / 大客户经理 / 跟单主管 / 生产计划主管 / 品管主管 / 样板主管 / 财务主管 / 招聘培训主管

  ⬇ 公司级事项（65 项）
    其中 16 项来自流程图节点（打样/货期/合同/...）
    其中 49 项来自行业模板（市场调研/客户开发/品质管控/...）

  ⬇ 部门内事项（4 部门 × 25 项 = 100 项）
    业务部 / 跟单部 / 研发部 / 生产部 各 25 项

  ⬇ KPI 范式（每岗位 4-6 个）
    总经理 KPI 5 个 / 业务总监 5 个 / 研发总监 6 个 / 生产总监 5 个 / ...

  ⬇ 产出表单（65 个）
    每条事项 1 个，如《市场调研报告》《订单评审纪要》《打样台账》《8D 报告》...

【灰色地带 · 8 项自动标 [问]】
  · 企业全称 / 简称（落盘目录用）           ← 必填
  · A 岗归属（具体人名）                    ← 留给现场访谈
  · 关键岗位实际任职者                      ← 留给现场访谈
  · 薪酬数字                                ← 永远 [问]
  · 历史包袱（外协人员、双工厂权责）        ← 留给现场访谈

═══════════════════════════════════════════════════════════
```

### 拍板问题

```json
{
  "question": "schema 草稿是否可以拍板生成？",
  "header": "拍板",
  "options": [
    {"label": "通过 → 立即生成 8 份产物 (Recommended)", "description": "30 秒后输出 computer:// 链接"},
    {"label": "调整字段", "description": "如：删某部门 / 加某岗位 / 改某事项 / 改 KPI 目标值"},
    {"label": "补充企业简称 + 任职者", "description": "把 [问] 项填回（最少补企业简称用于落盘）"},
    {"label": "重新跑（重新提供输入）", "description": "重新选行业 / 重新上传流程图"}
  ],
  "multiSelect": false
}
```

**最少必填**：企业简称（用于落盘目录命名）。

**最佳实践**：咨询师不需要现场填所有 [问] 项，因为这些是留给现场访谈的。生成 8 份产物时，[问] 项会原样写入「澄清说明.md」内部参考文件中。

## 2 轮版执行节奏总览

| 阶段 | 平均耗时 | 关键动作 |
|---|---|---|
| 第 0 轮（RAG 自动） | 30 秒 | 静默扫描官方案例 + 客户既有材料 |
| 第 1 轮 | **1-2 分钟** | 2 个 AskUserQuestion 一次性问完核心输入 |
| AI 自动推导 | 30 秒 - 1 分钟 | 解析流程图 + 加载模板 + 生成 schema 草稿 |
| 第 2 轮 | **3-7 分钟** | 咨询师 review schema：通过 / 调整 / 补充 |
| 生成 8 份产物 | 30 秒 | render 脚本输出 |
| **合计** | **5-10 分钟** | 比 v2.0.4 5 轮版（12-18 分钟）再压缩 50% |
