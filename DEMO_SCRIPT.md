# DEMO_SCRIPT.md — 2-3 分钟演示视频脚本

> 用于 GitHub Release / 官网首页 / 推广素材的产品演示视频脚本。
>
> 目标时长：**2 分 30 秒 - 3 分 00 秒**
> 目标观众：**组织咨询师 / 中小企业 HR / 咨询机构合伙人**
> 核心信息：**5 分钟产出 8 份组织设计交付件**（vs 传统 2-4 小时）

---

## 一、视频整体结构

| 分钟 | 段落 | 核心信息 |
|---|---|---|
| 0:00 - 0:15 | **痛点钩子** | "你是不是也在为组织诊断报告的格式劳动头疼？" |
| 0:15 - 0:45 | **方案展示** | 三张矩阵 + 工作分析表 + 预览，长什么样 |
| 0:45 - 1:30 | **使用演示** | 10 轮对话 5 分钟出图（屏幕录制） |
| 1:30 - 2:15 | **10 项特性快速过** | 字幕快闪 + 视觉化呈现 |
| 2:15 - 2:50 | **行动召唤** | GitHub 链接 + 安装命令 + 关注号召 |
| 2:50 - 3:00 | **片尾** | logo + 联系方式 |

## 二、详细分镜头脚本

### 镜头 1（0:00-0:15）· 痛点钩子

**画面**：
- 黑色背景，白色大字逐字出现
- 配上"打字机"音效

**字幕**：
```
组织设计的痛点是什么？

✋ 不是缺方法
✋ 不是缺案例
✋ 是格式劳动太重——
   10 部门 × 80 项 RACI
   15 张工作分析表
   每次 2-4 小时纯手工
```

**旁白**（女声 / 平稳）：
> "组织设计的痛，不在缺方法，不在缺案例，在于格式劳动太重。每做一家企业，10 部门 80 项 RACI、15 张工作分析表，纯手工 2 到 4 小时。"

### 镜头 2（0:15-0:45）· 方案展示

**画面**：
- 快闪展示 3 个产物：
  1. 表 1 部门分工矩阵.xlsx（10 部门 × 80 项，绿黄橙三色 RACI）
  2. 表 2 岗位分工矩阵.xlsx（按部门分 Sheet，R/A/I 字母）
  3. 表 3 工作分析表合集.docx（15 张表，每张 8 模块）
- 同时叠加 HTML 预览页（炫酷）

**字幕浮动**：
```
✨ 一次 10 轮对话
✨ 5 分钟产出 8 份文件
✨ 全程对齐官方案例格式
```

**旁白**：
> "现在，有了 org-design-toolkit，一次 10 轮对话、5 分钟，就能输出三张分工矩阵、15 张工作分析表、加上一份可视化预览，**全程对齐你的官方案例格式**。"

### 镜头 3（0:45-1:30）· 屏幕录制使用演示

**画面**：
- 真实屏幕录制（加快 2-3 倍速）：
  1. 在 Claude Code 中说："做一份示例企业的分工矩阵"
  2. AskUserQuestion 跳出来：企业简称、行业、是否有诊断
  3. 10 轮快速对话过场（部分加快）
  4. 进度条："生成中..."
  5. 8 份文件出现在工作目录
  6. 双击打开 xlsx，看到完整的部门分工矩阵

**字幕标注**：
- "第 0 步：自动 RAG 关联案例"
- "第 5 轮：客户产品流程图对接"
- "第 10 轮：预审 JSON schema"
- "✓ 7 道质量闸通过"
- "📂 8 份产物已落盘"

**旁白**（关键点提示）：
> "skill 启动后做的第一件事，是扫描你的工作目录，找到关联的官方案例。然后通过 10 轮对话采集信息——企业简称、行业、战略、架构、岗位、流程……每一轮都用 AskUserQuestion，不脑补、不胡填。最后 5 分钟出图。"

### 镜头 4（1:30-2:15）· 10 项特性快闪

**画面**：
- 每项 4 秒，配图标 + 简短文字
- 节奏感强（配快节奏背景音乐）

**10 项快闪**：

| # | 图标 | 标题 | 副标 |
|---|---|---|---|
| 1 | 🔍 | RAG 强制前置 | 对齐官方案例，不靠想象 |
| 2 | 📊 | 三张矩阵分表 | 部门 / 岗位 / 工作分析独立文件 |

| 4 | 📋 | 产出表单列必填 | 经验治理 → 流程治理的物理载体 |
| 5 | 🔄 | 流程驱动 | 按创利流程顺序，不按抽象主题 |
| 6 | 🔗 | 表间承上启下 | 岗位矩阵自动与部门矩阵一致 |
| 7 | 🏭 | 7+ 子行业模板 | 制造/精品包装/外贸/化工/食品/连锁/项目 |
| 8 | ⚡ | 三个专项动作 | 产品流程图 / 外协转内部 / 多地协同 |
| 9 | 📈 | 增量迭代 | v1→v2→v3 自动差异报告 |
| 10 | ✅ | 七道质量闸 | RACI/A 数/KPI/一致性/适配/产出/顺序 |

**旁白**：
> "10 项核心特性，每一项都来自真实项目踩坑沉淀。"

## 三、视觉风格指南

### 配色

- 主色：`#2F5597`（深蓝，专业感）
- 辅色：`#C6E0B4` 绿（主责）、`#FFE699` 黄（参与）、`#F4B084` 橙（监督）
- 背景：纯白 / 极浅灰 `#F5F7FB`
- 强调色：`#C00000`（红，用于错误 / 警告）

### 字体

- 标题：思源黑体 / Source Han Sans / 苹方 Bold
- 正文：苹方 / 微软雅黑 Regular
- 代码：JetBrains Mono / Menlo

### 动画风格

- 简洁，**不要花哨过渡**
- 字幕用淡入淡出（200ms）
- 屏幕录制部分用 2-3 倍速 + 关键帧停顿

### 音乐建议

- 背景音乐：低调电子 / lo-fi（音量调低至 -25dB）
- 推荐风格：tech / corporate / minimal
- 免费来源：YouTube Audio Library / Pixabay / Freesound
